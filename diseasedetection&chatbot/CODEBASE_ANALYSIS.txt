WHAT IS PLANT AI?
-----------------
Plant AI is a comprehensive, production-ready web application that combines 
AI-powered plant disease detection with an intelligent agricultural chatbot. 
The application is designed to help farmers identify plant diseases from 
images and get expert farming advice through a conversational AI interface.

KEY FEATURES:
- AI Disease Detection: Upload plant images for instant disease identification
  using state-of-the-art computer vision models (MobileNetV2)
- Smart Chatbot: Get expert farming advice from an AI assistant powered by 
  large language models (Mistral AI via OpenRouter)
- Voice Support: Speech-to-text and text-to-speech capabilities for hands-free
  operation using Google Speech Recognition and Google TTS
- Multilingual: Full support for English and Tamil (தமிழ்)
- Real-time Streaming: Responses stream in real-time using Server-Sent Events
- Responsive Design: Works on desktop, tablet, and mobile devices
- Dark Mode: Theme switching support

TARGET USERS:
- Farmers who need quick plant disease diagnosis
- Agricultural extension workers
- Home gardeners
- Agricultural students and researchers

FILE: chat.py (177 lines)
PREFIX: /api/v1/chat
PURPOSE: AI chatbot endpoints with streaming support

ENDPOINTS:

1. POST /chat
   - Description: Send chat message, get AI response
   - Input: ChatRequest body, X-Session-ID header (optional)
   - Process:
     a. Generate session ID if not provided
     b. Call llm_service.chat() with message, context, language
     c. Detect response language
   - Output: ChatResponse with response, language, timestamp

2. POST /chat/stream
   - Description: Stream chat response in real-time using SSE
   - Input: ChatRequest body, X-Session-ID header
   - Process:
     a. Create async generator
     b. Stream chunks via llm_service.chat_stream()
     c. Format as Server-Sent Events
   - Output: StreamingResponse with text/event-stream media type
   - Event Format: data: {"content": "chunk", "done": false}

3. DELETE /chat/history
   - Description: Clear chat history for a session
   - Input: X-Session-ID header (required)
   - Process: llm_service.clear_session(session_id)
   - Output: Success confirmation

4. GET /chat/languages
   - Description: Get supported chat languages
   - Output: List with code, name, native name for each language

SSE STREAMING IMPLEMENTATION:
```python
async def generate():
    async for chunk in llm_service.chat_stream(...):
        data = json.dumps({"content": chunk, "done": False})
        yield f"data: {data}\n\n"
    yield f"data: {json.dumps({'done': True})}\n\n"

return StreamingResponse(generate(), media_type="text/event-stream")
```

FILE: llm_service.py (479 lines)
CLASS: LLMService
PURPOSE: AI chatbot using OpenRouter API with Mistral AI

COMPONENTS:

1. ChatMemory Class:
   - In-memory chat history manager
   - Methods:
     * add_message(session_id, role, content): Add message to history
     * get_history(session_id): Get formatted history for API
     * get_context_string(session_id): Get history as string
     * clear_session(session_id): Remove session history
   - Max messages: 10 (configurable)

2. LLMService Initialization:
   - Configures OpenRouter API access
   - Sets up headers with auth and metadata
   - Initializes ChatMemory
   - Detects demo mode if API key not configured

3. System Prompts:
   - English Prompt: Expert agricultural advisor rules
     * Reply only in specified language
     * Concise responses (6-10 lines)
     * No markdown formatting
     * Complete sentences
     * Focus on organic solutions
     * Include safety precautions

   - Tamil Prompt: Same rules in Tamil language

4. Methods:

   chat(message, session_id, disease_context, language) -> str:
   - Sanitizes input
   - Auto-detects language if not specified
   - Builds context with disease info and chat history
   - Calls _get_complete_response() with smart continuation
   - Stores message in memory
   - Returns cleaned response

   chat_stream(message, session_id, disease_context, language) -> AsyncGenerator:
   - Same preparation as chat()
   - Streams response chunks via SSE
   - Updates memory when complete
   - Yields content chunks for real-time display

   _get_complete_response(prompt, language) -> str:
   - Makes API call to OpenRouter
   - Implements smart continuation logic:
     * Checks if response ends properly (sentence complete)
     * If not, requests continuation up to 4 times
     * Avoids repetition in continuations
   - Returns cleaned final response

   _detect_language(text) -> LanguageEnum:
   - Uses langdetect library
   - Returns TAMIL if "ta" detected, else ENGLISH

   _get_demo_response(message, disease_context, language) -> str:
   - Returns pre-defined demo responses
   - Different responses for English and Tamil
   - Includes setup instructions

OPENROUTER API CALL:

```python
payload = {

        {"role": "user", "content": user_prompt}
```

FILE: speech_service.py (222 lines)
CLASS: SpeechService
PURPOSE: Speech-to-text (STT) and text-to-speech (TTS) processing

COMPONENTS:

1. Initialization:
   - Configures temp directory for audio files
   - Initializes SpeechRecognition recognizer
   - Sets up language code mappings:
     * TTS: English -> "en", Tamil -> "ta"
     * STT: English -> "en-US", Tamil -> "ta-IN"

2. Methods:

   speech_to_text(audio_bytes, language) -> Tuple[str, LanguageEnum]:
   - Saves uploaded audio to temp file
   - Converts to WAV format using pydub AudioSegment
   - Uses SpeechRecognition with Google Speech API
   - If language not specified:
     * Tries Tamil first, then English
     * Returns detected language
   - Cleans up temp files
   - Returns: (transcribed_text, detected_language)

   text_to_speech(text, language) -> bytes:
   - Uses gTTS (Google Text-to-Speech)
   - Generates MP3 audio
   - Saves to temp file, reads bytes, returns
   - Cleans up temp file

   text_to_speech_stream(text, language) -> io.BytesIO:
   - Wrapper that returns BytesIO stream for streaming responses

   detect_language_from_text(text) -> LanguageEnum:
   - Uses langdetect library
   - Detects Tamil or defaults to English

AUDIO PROCESSING FLOW:
```python

audio = AudioSegment.from_file(temp_input)
```

```typescript
const reader = response.body?.getReader();
const decoder = new TextDecoder();

while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  
  const text = decoder.decode(value);
  const lines = text.split("\n");
  
  for (const line of lines) {
    if (line.startsWith("data: ")) {
      const data = JSON.parse(line.slice(6));
      if (data.content) onChunk(data.content);
      if (data.done) onComplete();
    }
  }
}
```
